HADES

Embedded Safety Monitoring System for Detecting Unattended Heat Sources

Ahmed Khan • Johnson Mac

Project Overview

HADES is an embedded monitoring system designed to detect potentially dangerous situations where a heat source remains active while no person is present to supervise it.

The system combines infrared temperature sensing with mmWave human presence detection to determine whether heat is being actively monitored by a user.

If a strong heat source is detected without nearby human presence for a sustained period of time, the controller can escalate the situation by sending notifications and activating a local alarm until the condition is acknowledged.

System Architecture

The HADES system is centered around the CC3200 LaunchPad microcontroller which acts as the primary control unit for sensing, processing, and device response.

Sensors communicate with the controller using I2C and UART interfaces while the OLED display is driven through SPI communication.

GPIO connections support alarm control and user input, allowing the system to combine sensing, display feedback, and local safety response.

System State Machine

The HADES controller operates using a structured state-machine workflow. After initialization, the system verifies connectivity and begins monitoring temperature and presence conditions.

If heat is detected while no human presence is reported, the controller begins tracking elapsed time for the condition and escalates the response when thresholds are exceeded.

Major Components

  • CC3200 LaunchPad Microcontroller
  • Infrared Temperature Sensor
  • mmWave Human Presence Radar
  • Adafruit OLED Display
  • Relay Control Module
  • Alarm Output Device
  • User Push Button
  • AWS Cloud Communication Interface
