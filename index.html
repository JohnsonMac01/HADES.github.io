HADES

Embedded Safety Monitoring System for Detecting Unattended Heat Sources

Overview Team Architecture State Machine Components Goals

Project Overview

HADES is an embedded monitoring system designed to detect potentially dangerous situations where a strong heat source remains active while no person is present to monitor it.

The system combines an infrared temperature sensor with a mmWave human presence sensor. By analyzing both heat and occupancy information, the system determines whether a potentially hazardous condition exists.

If heat is detected without nearby human presence for a sustained period of time, the controller can trigger notifications and activate local safety mechanisms.

System Architecture

The HADES system is centered around the CC3200 LaunchPad microcontroller which serves as the main processing unit. The controller interfaces with sensors, display hardware, alarm systems, and cloud communication services.

Sensors communicate through I2C and UART interfaces while the OLED display uses SPI. GPIO lines control the relay and alarm hardware, and a button input allows user acknowledgement or manual interaction with the system.

System State Machine

The HADES system operates using a structured state-machine workflow. After initialization, the controller verifies network connectivity and begins continuously monitoring temperature and presence conditions.

If heat is detected while no human presence is observed, a timer begins. If the condition persists beyond a threshold period, the system escalates to notification and alarm states until the situation is acknowledged.

Design Goals

The goal of this project is to develop an embedded safety system capable of detecting unattended heat sources before they become hazardous.

By combining environmental sensing, presence detection, and cloud communication, HADES demonstrates a complete IoT monitoring pipeline including sensing, decision-making, notification, and local response.
